Transações

O conceito de transação é fundamental em sistema de banco de dados, é considerada como uma "unidade de trabalho", que é realizada em qualquer sistema computacional de um modo coerente e independente de outras transações. Elas devem permitir que o sistema esteja num estado coerente antes e depois da sua execução, independente de falhas ou outros problemas que possam ocorrer. Devem permitir também que vários clientes diferentes acessem concorrentemente o sistema sem que isso possa corromper ou levar a estados que não sejam considerados coerentes.

A definição do conceito de transações envolve o acrônimo ACID, oriundo das propriedades de Atomicidade, Consistência, Isolamento e Durabilidade.

Atomicidade

Atomicidade: A propriedade de atomicidade em um banco de dados pressupõe que as operações realizadas numa transação sejam todas realizadas por completo ou que nenhuma seja realizada. Isso permite que, durante a nossa interação com um banco de dados, possamos agrupar vários comandos relacionados com a garantia de que todos sejam executados, de modo que as informações armazenadas permaneçam em um estado consistente após a execução da transação.

Vamos imaginar uma transferência de valores de uma conta bancária para outra. Primeiramente, o banco de dados vai ler a conta A, depois será sacado R$ 100,00 dessa conta, e então a operação de subtração será salva. Em seguida, será lida a conta B e creditado nela o valor de R$ 100,00, com a operação de crédito sendo salva. Note que são realizadas seis operações no total. Caso ocorra um erro em qualquer uma delas, o banco de dados voltará para seu estado inicial, cancelando a transferência.

Consistência

Consistência: Essa propriedade assegura que a execução de qualquer transação trará o banco de dados de um estado consistente para outro estado também consistente. Isso implica que todos os dados de um banco de dados devem ser válidos de acordo com um conjunto de regras que podem incluir restrições de tipo, valor e referência entre informações.

Usando o exemplo de transferências entre contas novamente, se antes da transferência o valor na conta A é de R$ 1000,00 e o valor na conta B é de R$ 500,00, a soma das duas contas no início da operação é de R$ 1500,00. Após a transferência (independente do valor transferido), a soma das duas contas deve permanecer R$ 1500,00, mantendo a consistência.

Isolamento

Mesmo existindo operações simultâneas o resultado final de todas as transações devem ser equivalentes.

Voltamos ao exemplo de transferência. Dentro de uma instituição bancária podem estar ocorrendo várias transferências ao mesmo tempo. No entanto, cada uma delas deve estar isolada das outras, garantindo a atomicidade e consistência das operações. O isolamento assegura que as operações de uma transação não afetem a execução de outras, mesmo quando são realizadas simultaneamente.

Durabilidade

A durabilidade garante que uma vez que uma transação tenha sido finalizada com sucesso, os dados terão a garantia de terem sido armazenados corretamente, independente da eventualidade de falhas, falta de energia, erros de aplicações entre outros.

Esta propriedade é crucial para a confiabilidade dos bancos de dados, tornando-os "ferramentas sagradas" para muitas empresas. Imagine o impacto de uma empresa perder todos os seus dados: a continuidade dos negócios estaria em risco. Mais importante do que os dados em si é o uso que se faz deles, pois a durabilidade assegura que essas informações vitais estarão sempre disponíveis e intactas.
