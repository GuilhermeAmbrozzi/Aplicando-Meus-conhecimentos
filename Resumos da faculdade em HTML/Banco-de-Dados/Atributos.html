Atributos

Os atributos são propriedades utilizadas para descrever uma entidade, podemos assim afirmar que os Atributos são características contidas nas Entidades. Por exemplo, em uma entidade Cliente, podemos relacionar os atributos CPF, NOME, IDADE, ENDEREÇO, BAIRRO E CIDADE

Outra maneira de demonstrar um Entidade e Atributos é em forma de Planilha. Verificamos que a tabela PRODUTOS contém atributos que são os nomes da colunas e um conceito novo chamado TUPLA de modo formal, uma linha é denominada de Tupla..

Tipos de Atributos

  • Atributo Simples: Contém um único valor para cada elemento da entidade. Nesse caso, pode ocorrer uma informação repetida. Para uma entidade Cliente, temos um nome para cada cliente podendo ocorrer de dois clientes com o mesmo nome, mas informando um dado para cada cliente. Segue exemplo da representação.
  • Atributo Multivalorado: Permite conter informações com diferentes valores. É uma solução do problema quando, você tem vários telefones para um Funcionário ou Cliente.. Segue exemplo, analise que no atributo telefone está com 2 cíngulos essa é a sua representação.
  • Atributo Composto: Nos permite indicar um atributo que pode ser dividido em outros. Um exemplo pode ser o ENDEREÇO, que podemos dividir em RUA, CIDADE, ESTADO e CEP.
  • Atributo Chave: Quando temos atributos de uma entidade, é importante sempre indicarmos um identificador, que podemos chamar também de Atributo-Chave. Esse atributo irá identificar o item da entidade de maneira única (sem repetições) no conjunto de elementos.

O Atributo-Chave deve ser íntegro, ou seja, sem repetições, e também não pode ser nulo. A sua representação pode ser de maneira sublinhada ou com um círculo destacado na borda em negrito.

Para entender de uma forma mais clara, para a entidade Funcionários, temos os seguintes atributos.

Endereço: Analisando esse atributo, sabemos que pode ocorrer de mais de um funcionário morando no mesmo endereço, logo, ele não pode ser classificado como um atributo identificador.

Nome: Esse atributo pode confundir você um pouco, pois cada funcionário tem seu nome, porém, pode haver funcionários com o mesmo nome, logo, podemos analisar que se usarmos ele como identificador pode ocorrer algum erro.

Matrícula: Para cada funcionário da empresa é gerado uma matrícula que serve para lhe identificar na empresa e que não pode se repetir. Esse atributo pode sim ser classificado como Atributo-Chave, devendo ser destacado entre os demais atributos.

Domínio

Para cada atributo criado devemos associar um tipo a ele. A essa associação, damos o nome de domínio. Segue alguns tipos de domínio.

  • Numéricos: inteiros e reais.
  • Caracteres
  • Booleanos
  • Cadeia de caracteres de tamanha fico e tamanha variável
  • Data
  • Hora

Com essas informações, iremos criar uma tabela chamada Funcionário, contendo NOME DO CAMPO, TIPOS e TAMANHO assim podemos ver de forma clara as aplicações dos tipos.

Chave Estrangeira

Trata-se de um campo na tabela que aponta para a chave primária de uma outra tabela.Assim trazendo uma relação entre duas entidades, seu objetivo é garantir a integridade dos dados referenciais, pois, nesses casos, serão permitidos valores que irão aparecer na base de dados. Vale relembrar que, após estabelecer uma chave estrangeira, o atributo marcado não permitirá a exclusão, inserção ou modificação de dados em tabelas que estejam dependentes uma da outra. Assim os administradores do banco de dados tem que ter uma maior atenção.

Integridade referencial é um conceito importante no uso de chaves estrangeiras. Ela garante que as relações entre tabelas permaneçam consistentes. Se um valor de chave estrangeira for alterado ou removido, a integridade referencial pode ser comprometida.

Restrição de Exclusão Não é permitido excluir uma entrada de uma tabela pai (como Departamento) se houver entradas na tabela filha (como Funcionário) que referenciem essa entrada. Isso é para evitar registros órfãos na tabela filha.

Exemplo: Se tentarmos excluir o departamento com Depto_Cod = 1 da tabela Funcionário, o banco de dados impedirá essa ação porque há funcionário na tabela Funcionário que ainda estão associado a esse departamento (João da Silva).

Restrição de Inserção Não é permitido inserir um valor na chave estrangeira que não existe na tabela pai. Isso garante que todos os valores referenciados na tabela filha sejam válidos.

Exemplo: Se tentarmos adicionar um novo departamento com Depto_Cod = 4 na tabela Funcionário, isso falhará porque não existe um departamento com Depto_Cod = 4 na tabela Departamento.

Restrição de Modificação Não é permitido modificar a chave primária na tabela pai se houver registros na tabela filha que dependam dela. Isso impede que as referências na tabela filha se tornem inválidas.

Exemplo: Se tentarmos alterar o Depto_Cod de 1 para 4 na tabela Departamento, o banco de dados não permitirá essa modificação, pois o funcionário João da Silva tem referencia Depto_Cod = 1.
