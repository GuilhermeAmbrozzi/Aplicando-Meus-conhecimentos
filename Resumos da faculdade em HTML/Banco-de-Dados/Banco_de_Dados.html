Introdução a Banco de Dados

Introdução

Na sociedade do século XXI, temos informação em abundância. O desafio dos próximos anos é encontrar formas eficientes de processar os dados que já temos para gerar conhecimento e riquezas.

Nos últimos anos, boa parte dos softwares desenvolvidos envolve mais do que processar informações, essas informações devem ser armazenadas em um mecanismo confiável e que venha possibilitar o acesso simples e rápido à leitura e escrita desses dados.

Por isso surgiu o termo "Banco de Dados" que, alguns anos atrás escrever sobre o tema seria uma tarefa relativamente tranquila, pois acreditavam tratar-se de um assunto absolutamente consolidado. Mas como podemos observar o nosso mundo está em constante mudança e com o surgimento de novos negócios, provocaram uma ruptura na forma de pensar de como essas informações devem ser armazenadas.

Por isso, quando pensamos sobre o significado de "Banco de Dados" logo, lembramos das instituições bancarias, que guardam de modo bastante seguro o nosso dinheiro. Os "bancos de dados" seriam, então, ferramentas que guardariam nossas informações (dados) de modo também supostamente seguro e confiável.

Entenda que um banco de dados é uma coleção de dados relacionados. Os dados são fatos que podem ser gravados e que possuem um significado implícito. Por exemplo, uma planilha de Excel, onde pode conter nomes, telefones, e-mails e endereços. Com o decorrer do tempo entraremos mais afundo no seu funcionamento e com isso uma melhor compreensão.

Características de Sistemas de Banco de Dados

Para entender um Banco de Dados e como ele nos ajuda, vamos analisar algumas características presentes em nosso cotidiano. Procure pensar em algumas aplicações que você utiliza e não fazem uso de um SGBD. Entre alguns exemplos, temos as planilhas,onde há a necessidade de armazenar informações manipuladas por meio de arquivos.

Em qualquer aplicação onde temos a necessidade do armazenamento de dados, é necessário dispor de um mecanismo que permita que esses sejam gravados de modo persistente, possibilitando que você os carregue eletrônica ou fisicamente para diferentes locais de maneira bastante simples. No entendo, uma desvantagem desse modelo é o trabalho envolvido na criação de um formato e no processamento da gravação e recuperação dos dados, e acredite, não é pouco trabalho!

Já em um Banco de Dados, temos à nossa disposição o SGBD que é um grupo de programas que trabalham em conjunto com o sistema operacional para criar, processar, armazenar, recuperar, controlar e gerenciar os dados. Desse modo, uma série de preocupações passa a ser delegada a um software. Vamos agora analisar uma série de características que diferenciam a abordagem de sistemas de banco de dados da manipulação manual das informações (como em arquivos, por exemplo).

Natureza auto descritiva: É uma característica fundamental que distingue os sistemas de banco de dados de outras abordagens: as metainformações sobre o banco de dados são armazenados em conjunto.

Nessa imagem, podemos observar a visão do professor que busca informações dos alunos em tabela salva no banco de dados. Ele encontrar dados como matrícula, nome, e-mail e idade.

Já nessa segunda foto, podemos observar as metainformações armazenadas no banco de dados, que incluem tipo, tamanho e restrições. Podemos ter uma PK como a matrícula do aluno, pois é o identificador único de um registro na tabela. Isso vai dependerá de cada projeto do banco de dados.


Isolamento entre Programa e Dados: Isso quer dizer que não tem uma dependência dos dados com os programas que controlam esses dados. Para um melhor entendimento, você pode modificar suas aplicações sem interfere nos dados armazenados. Além disso, se você modificar seus dados, também não interfere nas aplicações, pois são dois componentes separados, embora exista uma relação entre eles. Essa característica aumenta bastante o nível de manutenibilidade do sistema, quando bem aplicada.
